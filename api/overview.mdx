The WCA API gives developers programmatic access to competition data, results, person profiles, rankings, records, and more. All responses are JSON. Most read endpoints are public and require no authentication.

## Base URL

All API endpoints are served from:

```
https://www.worldcubeassociation.org/api/
```

Both API versions live under this base:

| Version | Prefix | Purpose |
|---------|--------|---------|
| v0 | `/api/v0/` | Stable, public API. Competitions, results, persons, rankings, records, search. |
| v1 | `/api/v1/` | Newer API focused on competition registrations and live results. Requires authentication. |

<Note>
  The rate limit for the public API is **60 requests per minute** per IP address. Requests from internal/private IP ranges (loopback, RFC 1918) are exempt. Exceeding this limit returns `429 Too Many Requests`.
</Note>

## Response Format

All responses are `application/json`. Successful responses return the requested data directly or wrapped in a named key (e.g. `{ "competition": {...} }` or `{ "result": [...] }`). Error responses always use the shape:

```json
{
  "error": "Human-readable error message"
}
```

For record-not-found errors, additional metadata is included:

```json
{
  "error": "Couldn't find Competition with 'id'=BadId",
  "data": {
    "model": "Competition",
    "id": "BadId"
  }
}
```

### Common HTTP Status Codes

| Status | Meaning |
|--------|---------|
| `200 OK` | Request succeeded |
| `400 Bad Request` | Missing or invalid query parameter |
| `401 Unauthorized` | No valid access token provided |
| `403 Forbidden` | Token is valid but lacks the required scope or permission |
| `404 Not Found` | The requested resource does not exist |
| `429 Too Many Requests` | Rate limit exceeded |

## Pagination

Endpoints that return collections are paginated. Use the `page` query parameter (1-indexed) and `per_page` to control page size. The default page size is **20** results (from `DEFAULT_API_RESULT_LIMIT` in the source). Pagination metadata is returned in the `Link` response header following the [RFC 5988](https://datatracker.ietf.org/doc/html/rfc5988) format used by Rails' `kaminari` pagination gem.

```bash
# Fetch page 2 of competitions, 25 per page
curl "https://www.worldcubeassociation.org/api/v0/competitions?page=2&per_page=25"
```

## Quick Start: Unauthenticated Request

Most read endpoints need no token. Here is a real request that fetches a competition by its WCA ID:

```bash
curl https://www.worldcubeassociation.org/api/v0/competitions/WC2025
```

```json
{
  "id": "WC2025",
  "name": "World Rubik's Cube Championship 2025",
  "venue": "...",
  "start_date": "2025-07-14",
  "end_date": "2025-07-20",
  "country_iso2": "AE",
  "event_ids": ["222", "333", "444", "555", "666", "777", "333bf", "333oh", "clock", "minx", "pyram", "skewb", "sq1", "444bf", "555bf", "333mbf"],
  "competitor_limit": 1500,
  ...
}
```

Fetch a person's profile by WCA ID:

```bash
curl https://www.worldcubeassociation.org/api/v0/persons/2003POCH01
```

## API v1 Endpoints

v1 is the newer API. It currently covers competition registrations and live results. All v1 requests require authentication.

<Note>
  v1 is not yet intended for general third-party use. Getting a JWT token currently requires a logged-in session on the website. The OAuth bearer token flow is the intended long-term path.
</Note>

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/competitions/:id/registrations` | List registrations for a competition |
| GET | `/api/v1/competitions/:id/registrations/:reg_id` | Single registration |
| POST | `/api/v1/competitions/:id/registrations` | Create a registration |
| PATCH | `/api/v1/competitions/:id/registrations/:reg_id` | Update a registration |
| PATCH | `/api/v1/competitions/:id/registrations/bulk_update` | Bulk-update registrations |
| PATCH | `/api/v1/competitions/:id/registrations/bulk_auto_accept` | Auto-accept waitlist entries |
| GET | `/api/v1/competitions/:id/registration_config` | Registration configuration |
| GET | `/api/v1/competitions/:id/live/rounds` | Live: round admin overview |
| GET | `/api/v1/competitions/:id/live/rounds/:round_id` | Live: results for a round |
| POST | `/api/v1/competitions/:id/live/rounds/:round_id` | Live: submit result |
| PATCH | `/api/v1/competitions/:id/live/rounds/:round_id` | Live: update result |
| PUT | `/api/v1/competitions/:id/live/rounds/:round_id/open` | Live: open a round |
| PUT | `/api/v1/competitions/:id/live/rounds/:round_id/clear` | Live: clear all results |
| GET | `/api/v1/competitions/:id/live/podiums` | Live: current podiums |
